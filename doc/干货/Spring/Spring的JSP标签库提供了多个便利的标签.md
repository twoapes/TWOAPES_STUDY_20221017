# Spring的JSP标签库中提供了多个便利的标签，还包括一些遗留的数据绑定标签

JSP标签|描述
--|--
&#60;s&#58;bind&#62;|将绑定属性的状态导出到一个名为status的页面作用域属性中，与&#60;s&#58;path&#62;组合使用获取绑定属性的值
&#60;s&#58;escapeBody&#62;|将标签体中的内容进行HTML和&#47;或JavaScript转义
&#60;s&#58;hasBindErrors&#62;|根据指定模型对象(在请求属性中)是否有绑定错误，有条件地渲染内容
&#60;s&#58;htmlEscape&#62;|为当前页面设置默认的HTML转义值
&#60;s&#58;message&#62;|根据给定的编码获取信息，然后要么进行渲染(默认行为)，要么将其设置为页面作用域、请求作用域、会话作用域或应用作用域的变量(通过使用var和scope属性实现)
&#60;s&#58;nestedPath&#62;|设置嵌入式的path,用于&#60;s&#58;bind&#62;之中
&#60;s&#58;theme&#62;|根据给定的编码获取主题信息，然后要么进行渲染(默认行为)，要么将其设置为页面作用域、请求作用域、会话作用域或应用作用域的变量(通过使用vr和scope属性实现)
&#60;s&#58;transform&#62;|使用命令对象的属性编辑器转换命令对象中不包含的属性
&#60;s&#58;url&#62;|创建相对于上下文的URL,支特URl模板变量以及HTML&#47;XML&#47;JavaScript转义。可以渲染URL(默认行为)，也可以将其设置为页面作用域、请求作用域、会话作用域或应用作用域的变量(通过使用var和scope属性实现)
&#60;s&#58;eval&#62;|计算符合Spring表达式语言(Spring Expression Language,SpEL)语法的某个表达式的值，然后要么进行渲染(默认行为)，要么将其设置为页面作用域、请求作用域会话作用域或应用作用域的变量(通过使用var和scope属性实现)
